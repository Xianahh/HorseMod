# HorseMod AnimSet and Animation Variables documentation

This document describes all the XML animations in `42/media/AnimSets` for the horse (buck) and rider (player).

> [!WARNING]
> This file is outdated.

## Variable reference (Lua usage)
| Variable | Purpose in Lua | Key locations | File(s) |
| --- | --- | --- | --- |
| `isHorse` | Set when an IsoAnimal is recognised as a horse so the horse-only AnimSets can activate. | Horse initialisation in `HorseManager.initialiseHorse()` sets the flag when horses are discovered.| `42/media/lua/shared/HorseMod/HorseManager.lua` |
| `animalWalking` / `animalRunning` | Updated from rider input to drive horse locomotion states and transitions between walk/trot/run. | `MountController:update()` toggles these on the mount based on movement and run input, and clears them when stopped.| `42/media/lua/client/HorseMod/mount/MountController.lua` |
| `HorseGallop` / `HorseTrot` | Animation variables toggled from player input to select gallop or trot movement sets. | Gallop is mirrored from rider input in `MountController:update()`; trot is toggled via the rider in `Riding.toggleTrot()` and mirrored to the mount pair.| `42/media/lua/client/HorseMod/mount/MountController.lua`<br> `42/media/lua/client/HorseMod/Riding.lua` |
| `IdleToRun` / `idleToRun` | Used to request an idle→run blend; mirrored between rider and mount. | Mirrored from rider to mount in `MountController:update()` to keep AnimSet variables aligned. |`42/media/lua/client/HorseMod/mount/MountController.lua` |
| `targetTwist` / turning flags (`isTurningLeft`, `isTurningRight`, `isTurningAround`) | Directional steering from the engine; mirrored onto the rider so both animations pick the same turning nodes. | Turning flags are copied from the mount to the rider in `MountController:update()` so the player AnimSets see the horse’s turn state.| `42/media/lua/client/HorseMod/mount/MountController.lua` |
| `HorseJump` | Toggled when the rider presses the configured jump key to let the horse pathfinding AnimSets handle fences. | `Riding.handleKey()` sets `HorseJump` on the mount pair; `MountController.blockedBetween()` checks it when deciding whether fences are passable.| `42/media/lua/client/HorseMod/Riding.lua`<br> `42/media/lua/client/HorseMod/mount/MountController.lua` |
| `RidingHorse` | Marks the rider as mounted so rider AnimSets switch to mounted locomotion and disable normal combat. | Set during mount creation and cleared on cleanup in `Mount`; also used in mounted kick handling to disable ground attacks.|`42/media/lua/client/HorseMod/mount/Mount.lua`<br>`42/media/lua/client/HorseMod/mount/Mount.lua`<br>`42/media/lua/client/HorseMod/player/MountedKick.lua` |
| `HasReins` | Mirrors whether reins are equipped, selecting rein-specific turning variants. | Updated in `MountController:updateReins()` depending on available attachment data.|`42/media/lua/client/HorseMod/mount/MountController.lua` |
| `HorseEating` / `HorseEatingHand` | Drives eating idles and sound triggers when the horse grazes or is hand-fed. | Feeding timed action toggles `HorseEatingHand`; audio logic checks `HorseEating` to play chewing sounds.|`42/media/lua/client/HorseMod/player/ISFeedFromHand.lua`<br>`42/media/lua/client/HorseMod/Sounds.lua` |
| `HorseDying` / `HorseDead` / `HorseDeath` | Flags for death and downed states used by both mount and rider AnimSets to exit riding. | Set during death/fall animations and read by mounted state handling to lock movement and dismount safely.|`42/media/lua/client/HorseMod/mount/Mount.lua`<br>`42/media/lua/client/HorseMod/player/DismountAction.lua` |
| `kickLeft` / `kickRight` / `idleKicking` / `moveKicking` | Drive mounted kicking animations and enforce cooldown. | Mounted kick handler toggles these variables based on zombie position and clears them after the cooldown expires.|`42/media/lua/client/HorseMod/player/MountedKick.lua` |
| `MountFinished` / `DismountFinished` / `MountingHorse` / `DismountStarted` / `EquipFinished` / `EquippingSaddle` | Flow-control variables for mount/dismount/equip timed actions that advance AnimSet transitions. | Each timed action sets or clears the corresponding variable while playing its animation to let AnimSets transition back to idle.|`42/media/lua/client/HorseMod/player/MountAction.lua`<br>`42/media/lua/client/HorseMod/player/DismountAction.lua`<br>`42/media/lua/shared/HorseMod/TimedActions/ISHorseEquipGear.lua` |
| `TrailerAnimation`, `idleAction`, `eatingAnim`, `bMoving`, etc. | Contextual values provided by behaviours (e.g., trailer idling, idle action choices). | These values come from AI/behaviour scripts and are surfaced directly through the AnimSet conditions; they are not explicitly set in the Lua listed above but are consumed by the engine to pick matching nodes.|`42/media/AnimSets/buck/trailer/Horse_Trailer.xml` |
